Software Development

At NSL we are constantly writing and maintaining software for seismic data acquisition, earthquake detection and analysis, data archiving, fire camera image processing, networking, monitoring, and countless other projects related to managing environmental sensor networks. Our source code is available to the extent the law and our staffing allows, with the hopes that the environmental fields can more closely work together to maintain professional operational software. This site is an overview of projects based on general category. It is a preliminary work-in-progress, stay tuned for more!

Projects

Data Formats

We have an extensive collection of software utilities for data conversion and encoding/decoding to and from various formats that are only used in seismology, like QuakeML formatting in qmlutil, Steim compression decoding for Go and Python, etc (the fact that seismology has its own formats and we should abandon them as soon as humanly possible), and maybe our RefTek refutil packet-processing libraries

Analysis

We have projects that enable seismic analysis and research, like HASHpy, a Fortran/Python rewrite of the HASH first-motion focal mechanism program, and implementations of published algorithms, like the eqclustering method. We also have code that integrates with 3rd party services like the USGS Product Description Layer (PDL) and FindMeSpot.

Visualization

Description of projects and forks that enable visualization of data and products, like our focal mech stereonet.js, maybe some mapping stuff, maybe not.

Infrastructure

Description of projects to manage installation and dependencies of seismic software, systemd files, Salt states and formulas for using Antelope, ObsPy, Ringservers, RefTek utilities, SeisComp3, and assets in cloud providers, with and without Salt, i.e. Lambda scripts or GCP queries?
